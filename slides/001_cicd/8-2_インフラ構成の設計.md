# このレクチャーで学ぶこと

- リソースの設計

- 環境変数の設計

---

# 必要なリソース一覧

| リソース | 役割 |
|----------|------|
| SNS Topic | 通知の仲介 |
| Lambda Function | メッセージ変換 |
| IAM Role | Lambda権限管理 |
| CloudWatch Logs | ログ保存 |

---

# SNS Topicの設計

- **トピック名**: CloudWatchAlarmTopic
- Lambdaをサブスクライバーとして登録
- CloudWatch Alarmからメッセージを受信

---

# Lambda Functionの設計

| 項目 | 値 |
|------|-----|
| 関数名 | SlackNotificationFunction |
| ランタイム | Node.js 20.x |
| メモリ | 128MB |
| タイムアウト | 10秒 |

---

# データの流れ

```
CloudWatch Alarm
    ↓ SNSにメッセージ送信
SNS Topic
    ↓ Lambdaを起動
Lambda Function
    ↓ メッセージをパース
    ↓ Slack形式に変換
    ↓ Webhook URLにPOST
Slack
```

---

# SNSメッセージの構造

```javascript
{
  "Records": [{
    "Sns": {
      "Message": "{\"AlarmName\":\"...\", ...}"
    }
  }]
}
```

Messageは文字列化されたJSON

---

# CloudWatch Alarmメッセージ

```json
{
  "AlarmName": "HighCPUUtilization",
  "NewStateValue": "ALARM",
  "NewStateReason": "Threshold exceeded",
  "StateChangeTime": "2024-01-15T10:30:00Z"
}
```

---

# Slackメッセージの構造

```json
{
  "text": "🚨 AWS Alert",
  "attachments": [{
    "color": "danger",
    "fields": [
      { "title": "Alarm", "value": "..." },
      { "title": "Status", "value": "..." }
    ]
  }]
}
```

---

# IAM権限の設計

**Lambda基本実行ロール**

- CloudWatch Logsへの書き込み
- SAMで自動アタッチ

```yaml
Policies:
  - AWSLambdaBasicExecutionRole
```

---

# 環境変数の設計

| 変数名 | 用途 |
|--------|------|
| SLACK_WEBHOOK_URL | Slack通知先 |

コードに直接書かない → セキュリティ確保

---

# エラーハンドリング

- SNSメッセージのパースエラーをキャッチ
- Slackへのリクエストエラーをキャッチ
- エラー時はCloudWatch Logsに記録
- リトライはSNSが自動で実行

---

# ログ設計

- CloudWatch Logsに実行ログを保存
- ログレベル: INFO, ERROR, DEBUG
- 本番ではDEBUGを無効化
- ログ保存期間を設定

---

# まとめ

- SNS Topicで通知を仲介
- Lambdaでメッセージを変換
- 環境変数でWebhook URLを管理
- IAM権限は最小限に

次のレクチャーでSAMテンプレートを実装します
