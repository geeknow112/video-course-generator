---

# ソースステージのエラー①

**接続エラー**

```
Connection is not available
```

**原因**: GitHub接続が保留中/削除された

**対処**: 接続を再作成、または認証をやり直す

---

# ソースステージのエラー②

**リポジトリが見つからない**

```
Repository not found
```

**原因**: リポジトリ名のスペルミス/削除された

**対処**: リポジトリ名を確認、接続の権限を確認

---

# ソースステージのエラー③

**ブランチが見つからない**

```
Branch not found
```

**原因**: ブランチ名のスペルミス/削除された

**対処**: ブランチ名を確認、mainブランチが存在するか確認

---

# デプロイステージのエラー①

**バケットが見つからない**

```
Bucket does not exist
```

**原因**: バケットが削除された/リージョンが違う

**対処**: バケットを再作成、リージョンを確認

---

# デプロイステージのエラー②

**アクセス拒否**

```
Access Denied
```

**原因**: S3バケットポリシー/IAMロールの権限不足

**対処**: バケットポリシーを確認、サービスロールの権限を確認

---

# 権限エラー

CodePipelineサービスロールに必要な権限:

- S3へのアクセス権限
- GitHub接続の使用権限
- CloudWatch Logsへの出力権限

---

# エラー詳細の確認方法

1. 失敗したステージをクリック
2. 「詳細を表示」をクリック
3. エラーメッセージを確認
4. CloudWatch Logsでログを確認

---

# 再実行の方法

**「変更をリリースする」**
→ 最新のソースで再実行

**「失敗したステージを再試行」**
→ 失敗したステージから再開

---

# デバッグのコツ

1. **エラーメッセージをよく読む**
2. **IAMロールの権限を確認**
3. **CloudWatch Logsで詳細ログを確認**
4. **AWSドキュメントを参照**

---

# パイプラインの編集

1. パイプラインを選択
2. 「編集」をクリック
3. ステージ/アクションを追加・変更・削除
4. 「変更を保存」

---

# 実践練習

わざとエラーを起こしてみましょう：

1. リポジトリ名を間違えてみる
2. バケット名を間違えてみる
3. エラーメッセージを確認
4. 修復してみる

---

# まとめ

- エラーメッセージをよく読むことが大切
- 権限エラーが最も多い
- 再実行で解決することも多い
- CloudWatch Logsで詳細を確認
