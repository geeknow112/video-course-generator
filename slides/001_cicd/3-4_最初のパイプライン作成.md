# 最初のパイプライン作成

AWS CI/CDパイプライン構築マスター講座
セクション3 - レクチャー4

---

# このレクチャーで学ぶこと

- パイプラインの作成手順
- ソースステージの設定
- デプロイステージの設定
- 初回実行の確認

---

# 作成するパイプライン

```
GitHub → S3
（ソース）  （デプロイ）
```

最もシンプルな構成から始めます

---

# パイプライン作成の開始

1. CodePipelineコンソールを開く
2. 「パイプラインを作成」をクリック
3. 「カスタムパイプラインを構築」を選択

---

# ステップ1: パイプライン設定

| 項目 | 設定値 |
|------|--------|
| パイプライン名 | demo-pipeline |
| パイプラインタイプ | V2 |
| サービスロール | 新しいサービスロール |

---

# パイプライン名のルール

- 英数字とハイフンのみ
- 100文字以内
- リージョン内で一意

```
良い例: demo-pipeline, my-cicd-pipeline
悪い例: demo_pipeline, Demo Pipeline
```

---

# ステップ2: ソースステージ

| 項目 | 設定値 |
|------|--------|
| ソースプロバイダー | GitHub (Version 2) |
| 接続 | 作成した接続を選択 |
| リポジトリ名 | cicd-demo-site |
| ブランチ名 | main |

---

# トリガーの設定

**「プッシュでパイプラインを開始」**

- mainブランチへのプッシュで自動実行
- Webhookが自動設定される

---

# ステップ3: ビルドステージ

今回は**スキップ**します

- 静的サイトなのでビルド不要
- 次のセクションでCodeBuildを追加

---

# ステップ4: デプロイステージ

| 項目 | 設定値 |
|------|--------|
| デプロイプロバイダー | Amazon S3 |
| バケット | セクション2で作成したバケット |
| デプロイ前にファイルを抽出 | チェックを入れる |

---

# 「ファイルを抽出」とは

**チェックあり**: ZIPを展開してアップロード
```
index.html
style.css
```

**チェックなし**: ZIPのままアップロード
```
source.zip
```

静的サイトは**チェックあり**が必要

---

# 確認画面

設定内容を確認して「パイプラインを作成」

- パイプライン名
- ソース: GitHub
- デプロイ: S3

---

# 初回実行

パイプライン作成後、**自動的に初回実行**が開始

```
Source（進行中）→ Deploy（待機中）
```

数分で完了します

---

# 実行結果の確認

**成功の場合**
- 各ステージが緑色で「成功」
- S3にファイルがアップロードされている

**失敗の場合**
- 赤色で「失敗」
- 詳細をクリックしてエラーを確認

---

# S3での確認

1. S3コンソールを開く
2. バケットを選択
3. ファイルが存在するか確認
4. ウェブサイトURLにアクセス

---

# まとめ

- シンプルなGitHub→S3パイプラインを作成
- ソースとデプロイの2ステージ構成
- 初回実行が自動で開始される

次のレクチャーでトラブルシューティングを学びます
