# プロジェクトの目的

**CloudWatchアラームをSlackに通知**

- AWSリソースの監視
- 異常検知時に即座に通知
- CI/CDで自動デプロイ

---

# なぜこのプロジェクトか

**実務で使える構成**

- 監視は運用の基本
- サーバーレスで低コスト
- CI/CDの総まとめ

---

# システム構成図

```
CloudWatch Alarm
    ↓
SNS Topic
    ↓
Lambda Function
    ↓
Slack Webhook
```

---

# 使用するAWSサービス

| サービス | 役割 |
|----------|------|
| CloudWatch | メトリクス監視 |
| SNS | 通知の仲介 |
| Lambda | メッセージ変換 |
| SAM | インフラ定義 |
| CodePipeline | 自動デプロイ |

---

# 完成イメージ

**EC2のCPU使用率が80%を超えたら...**

1. CloudWatchがアラーム発火
2. SNSがLambdaを起動
3. LambdaがSlackに通知
4. Slackにメッセージが届く

---

# Slack通知の例

```
🚨 AWS Alert
Alarm: HighCPUUtilization
Status: ALARM
Instance: i-1234567890abcdef0
Reason: CPU > 80%
Time: 2024-01-15 10:30:00 UTC
```
