# 静的ウェブサイトホスティングの設定

〜S3でウェブサイトを公開しよう〜

---

# このレクチャーでやること

1. 静的ウェブサイトホスティングを有効化
2. インデックスドキュメントを設定
3. パブリックアクセスを許可
4. バケットポリシーを設定

---

# 静的ウェブサイトホスティングを有効化

1. S3コンソールでバケットを選択
2. 「プロパティ」タブをクリック
3. 「静的ウェブサイトホスティング」→「編集」
4. 「有効にする」を選択

---

# インデックスドキュメントの設定

**インデックスドキュメント**: `index.html`

- サイトのトップページになるファイル
- URLの末尾が `/` の場合に表示される

**エラードキュメント**（任意）: `error.html`
- 404エラー時に表示されるページ

---

# エンドポイントURL

設定を保存すると、エンドポイントURLが発行されます

```
http://バケット名.s3-website-ap-northeast-1.amazonaws.com
```

このURLでウェブサイトにアクセスできます
（まだファイルがないのでエラーになります）

---

# パブリックアクセスの許可

1. 「アクセス許可」タブをクリック
2. 「ブロックパブリックアクセス」→「編集」
3. **すべてのチェックを外す**
4. 「変更の保存」→ 確認で「確認」と入力

**注意**: 公開設定なので、機密情報は置かないこと

---

# バケットポリシーの設定

「アクセス許可」→「バケットポリシー」→「編集」

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::バケット名/*"
    }
  ]
}
```

**バケット名を自分のものに置き換えてください**

---

# バケットポリシーの意味

- **Effect: Allow** - 許可する
- **Principal: \*** - 誰でも
- **Action: s3:GetObject** - ファイルの取得
- **Resource** - 対象のバケット内すべて

→ 「誰でもこのバケットのファイルを取得できる」

---

# 設定完了の確認

- 「アクセス」が「公開」になっている
- エンドポイントURLが発行されている
- バケットポリシーが設定されている

---

# まとめ

- 静的ウェブサイトホスティングを **有効化**
- インデックスドキュメントに **index.html** を設定
- パブリックアクセスの **ブロックを解除**
- バケットポリシーで **公開設定**

**次のレクチャー → HTMLファイルをアップロード**
