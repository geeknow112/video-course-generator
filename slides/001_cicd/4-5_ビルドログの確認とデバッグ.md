AWS CI/CDパイプライン構築マスター講座
セクション4 - レクチャー5

---

# このレクチャーで学ぶこと

- ビルドログの確認方法
- よくあるエラーと対処法
- デバッグのコツ
- ローカルでのテスト方法

---

# ビルドログの確認方法

1. CodeBuildコンソールを開く
2. ビルドプロジェクトを選択
3. ビルド履歴から対象を選択
4. 「ビルドログ」タブを確認

---

# ログの構造

```
[Container] Phase: INSTALL
[Container] Running command npm install
...
[Container] Phase: BUILD
[Container] Running command npm run build
...
[Container] Phase complete: BUILD State: SUCCEEDED
```

---

# フェーズごとの確認

| フェーズ | 確認ポイント |
|---------|-------------|
| INSTALL | 依存関係のインストール |
| PRE_BUILD | テスト結果 |
| BUILD | ビルド出力 |
| POST_BUILD | デプロイ結果 |

---

# よくあるエラー①

**npm install 失敗**

```
npm ERR! code ERESOLVE
npm ERR! ERESOLVE unable to resolve dependency tree
```

**対処**: package-lock.json を削除して再生成

---

# よくあるエラー②

**権限エラー**

```
AccessDenied: Access Denied
```

**対処**: IAMロールの権限を確認

---

# よくあるエラー③

**ファイルが見つからない**

```
ENOENT: no such file or directory
```

**対処**: パスの確認、artifacts設定の確認

---

# よくあるエラー④

**タイムアウト**

```
Build timed out
```

**対処**: タイムアウト値を延長、または処理を最適化

---

# デバッグのコツ

1. **エラーメッセージをよく読む**
2. **どのフェーズで失敗したか確認**
3. **echoでデバッグ出力を追加**
4. **ローカルで再現テスト**

---

# echoでデバッグ

```yaml
phases:
  build:
    commands:
      - echo "Current directory:"
      - pwd
      - echo "Files:"
      - ls -la
      - npm run build
```

---

# CloudWatch Logsとの連携

- ビルドログはCloudWatch Logsに保存
- 長期保存が可能
- ログインサイトで検索可能

---

# ローカルでのテスト

**CodeBuild Local Agent**

```bash
# Dockerが必要
docker pull amazon/aws-codebuild-local

# ローカルでビルド実行
./codebuild_build.sh -i amazon/aws-codebuild-local
```

---

# 再ビルドの方法

1. CodeBuildコンソールで「ビルドを再試行」
2. または「ビルドの開始」で新規実行
3. パイプラインから「変更をリリース」

---

# まとめ

- ビルドログでエラー原因を特定
- フェーズごとに確認ポイントが異なる
- echoでデバッグ出力を追加
- ローカルテストで効率的にデバッグ
