# テスト失敗時のパイプライン停止

AWS CI/CDパイプライン構築マスター講座
セクション5 - レクチャー4

---

# このレクチャーで学ぶこと

- 統合buildspec.ymlの作成
- テスト失敗時の動作確認
- 実践的なCI/CDフロー

---

# 統合buildspec.yml

```yaml
version: 0.2
phases:
  install:
    runtime-versions:
      nodejs: 18
      php: 8.2
    commands:
      - cd js && npm install
      - cd ../php && composer install
```

---

# 統合buildspec.yml（続き）

```yaml
  pre_build:
    commands:
      - echo "Running Jest tests..."
      - cd js && npm test
      - echo "Running PHPUnit tests..."
      - cd ../php && ./vendor/bin/phpunit
  build:
    commands:
      - echo "All tests passed!"
```

---

# テスト実行の流れ

```
Jest テスト
    ↓ 成功
PHPUnit テスト
    ↓ 成功
ビルド処理
    ↓
デプロイ
```

---

# テスト失敗時の動作

```
Jest テスト
    ↓ 失敗！
パイプライン停止
（PHPUnitは実行されない）
（デプロイされない）
```

---

# なぜ停止するのか

**exit code の仕組み**

- テスト成功: exit code 0
- テスト失敗: exit code 1（0以外）

CodeBuildは exit code 0 以外で**失敗**と判定

---

# 実験：わざと失敗させる

```javascript
// sum.test.js
test('わざと失敗', () => {
  expect(1 + 1).toBe(3);  // 失敗する
});
```

---

# 失敗時のログ

```
[Container] Running command npm test

FAIL ./sum.test.js
  ✕ わざと失敗

[Container] Command did not exit successfully
[Container] Phase complete: PRE_BUILD State: FAILED
```

---

# パイプラインの状態

```
Source: 成功 ✓
Build:  失敗 ✗  ← ここで停止
Deploy: 実行されない
```

**不完全なコードはデプロイされない**

---

# 修正して再実行

1. テストを修正
2. GitHubにプッシュ
3. パイプラインが自動実行
4. テスト成功 → デプロイ

---

# ベストプラクティス

- **ローカルでテスト**してからプッシュ
- **小さな変更**を頻繁にコミット
- **テストカバレッジ**を意識
- **失敗したらすぐ修正**

---

# 完成したプロジェクト構成

```
project/
├── js/
│   ├── package.json
│   ├── sum.js
│   └── sum.test.js
├── php/
│   ├── composer.json
│   ├── src/Calculator.php
│   └── tests/CalculatorTest.php
└── buildspec.yml
```

---

# まとめ

- 複数言語のテストを1つのbuildspecで実行
- テスト失敗でパイプラインが停止
- 不完全なコードはデプロイされない
- これがCI/CDの本質

これでセクション5は終了です
