# 動作確認とテスト

AWS CI/CDパイプライン構築マスター講座
セクション8 - レクチャー5

---

# このレクチャーで学ぶこと

- パイプラインの実行確認
- CloudWatch Alarmの設定
- テストアラームの発火
- Slack通知の確認

---

# パイプラインの実行確認

1. GitHubにプッシュ
2. CodePipelineコンソールを開く
3. パイプラインが実行中になる
4. 各ステージの状態を確認

---

# ソースステージの確認

- GitHubからの取得が成功したか確認
- 失敗した場合:
  - 接続設定を確認
  - ブランチ名が正しいか確認

---

# ビルドステージの確認

- CodeBuildのログを確認
- `sam build` が成功したか
- `sam package` が成功したか
- エラーがあればログを確認

---

# デプロイステージの確認

- CloudFormationコンソールを開く
- スタックの状態を確認
- **CREATE_COMPLETE** なら成功
- 失敗した場合はイベントを確認

---

# 作成されたリソースの確認

| リソース | 確認場所 |
|----------|----------|
| Lambda関数 | Lambdaコンソール |
| SNS Topic | SNSコンソール |
| 環境変数 | Lambda設定タブ |

---

# CloudWatch Alarmの設定

1. CloudWatchコンソールを開く
2. 「アラームを作成」をクリック
3. メトリクスを選択
4. しきい値を設定
5. アクションにSNS Topicを指定

---

# テスト用アラームの作成

**すぐに発火するアラーム**

- メトリクス: EC2 CPUUtilization
- しきい値: 1%以上
- アクション: 作成したSNS Topic

---

# アラームの発火確認

1. CloudWatchコンソールでアラーム状態を確認
2. **ALARM** になっていれば発火
3. SNSコンソールでメッセージ送信を確認

---

# Slack通知の確認

1. Slackを開く
2. 指定したチャンネルを確認
3. アラートメッセージが届いていれば成功！

```
🚨 AWS Alert
Alarm: TestHighCPU
Status: ALARM
```

---

# トラブルシューティング

**Slackに通知が来ない場合**

1. CloudWatch Logsを確認
2. Lambdaの実行ログを確認
3. Webhook URLが正しいか確認
4. IAMロールの権限を確認

---

# よくあるエラーと対処

| エラー | 対処法 |
|--------|--------|
| Lambdaタイムアウト | タイムアウト値を増加 |
| Webhook URLエラー | URLを再確認 |
| 権限エラー | IAMロールを確認 |

---

# テスト完了後の整理

- テスト用アラームを削除
- 本番用アラームを作成
- 適切なしきい値を設定
- 必要なメトリクスを監視

---

# まとめ

- パイプラインが成功
- CloudWatch Alarmを設定
- Slackに通知が届いた
- 実践的な監視システムが完成！
