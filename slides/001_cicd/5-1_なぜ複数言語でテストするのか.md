# 現実のプロジェクト

多くのプロジェクトは**複数の技術**で構成

- フロントエンド: JavaScript/TypeScript
- バックエンド: PHP, Python, Java
- インフラ: Terraform, CloudFormation

---

# よくある構成例

```
プロジェクト/
├── frontend/     ← React (Jest)
├── backend/      ← PHP (PHPUnit)
├── api/          ← Python (pytest)
└── infra/        ← Terraform
```

---

# 複数言語テストのメリット

1. **全体の品質担保**: 各コンポーネントをテスト
2. **早期発見**: どこで問題が起きたか特定
3. **チーム間連携**: 共通のCI/CDパイプライン

---

# CodeBuildの強み

**1つのビルドで複数言語をテスト可能**

```yaml
runtime-versions:
  nodejs: 18
  php: 8.2
  python: 3.11
```

---

# このセクションで作るもの

```
テストプロジェクト/
├── js/
│   └── sum.test.js      ← Jest
├── php/
│   └── CalculatorTest.php  ← PHPUnit
└── buildspec.yml
```

---

# パイプラインの流れ

```
ソース取得
    ↓
Jest テスト実行
    ↓
PHPUnit テスト実行
    ↓
全て成功 → デプロイ
1つでも失敗 → 停止
```

---

# 学習のポイント

- 複数のテストフレームワークを1つのbuildspecで実行
- テスト失敗時のパイプライン停止
- 実践的なCI/CD構成
