# AWSのCI/CDサービス概要

〜このコースで使うサービスを把握しよう〜

---

# AWSのCI/CDサービス一覧

| サービス | 役割 |
|---------|------|
| **CodePipeline** | パイプライン管理（司令塔） |
| **CodeBuild** | ビルド・テスト実行 |
| **S3** | 静的サイトホスティング |
| **Lambda** | サーバーレス関数 |
| **SAM** | サーバーレスアプリ定義 |

※ソースコード管理は **GitHub** を使用

---

# CodePipeline

**CI/CDパイプラインの司令塔**

✅ パイプライン全体の制御
✅ 各ステージの連携
✅ 実行状況の可視化
✅ 複数のソース・デプロイ先に対応

**料金**: 月額 $1/パイプライン（最初の1つは無料）

---

# CodeBuild

**ビルドとテストを実行するサービス**

✅ サーバー管理不要（フルマネージド）
✅ 複数の言語・ランタイムに対応
✅ buildspec.yml で設定
✅ 並列ビルドが可能

**料金**: 月100分まで無料

---

# S3

**オブジェクトストレージサービス**

このコースでの用途：
- 静的ウェブサイトのホスティング
- ビルドアーティファクトの保存

✅ 高い耐久性（99.999999999%）
✅ 低コスト（約 $0.025/GB/月）

---

# Lambda

**サーバーレスコンピューティング**

このコースでの用途：
- サーバーレス関数のデプロイ先
- イベント駆動の処理

✅ サーバー管理不要
✅ 実行時間に応じた課金
✅ 月100万リクエストまで無料

---

# SAM

**Serverless Application Model**

サーバーレスアプリを定義するフレームワーク

✅ YAMLでインフラを定義
✅ Lambda, API Gateway, DynamoDB を簡単に定義
✅ SAM CLIでローカルテスト可能

**SAM自体は無料**

---

# サービスの関係図

```
GitHub ──→ CodePipeline ──→ CodeBuild ──→ S3 / Lambda
              │                │
              │                └── buildspec.yml
              │
              └── パイプライン定義（SAMテンプレート）
```

**CodePipeline** が全体を制御
**CodeBuild** がビルド・テストを担当
最終的に **S3** や **Lambda** にデプロイ

---
