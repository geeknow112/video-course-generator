# CodeStar Connectionsとは

**外部サービスとの接続を管理**

- GitHub、GitLab、Bitbucketに対応
- OAuth認証で安全に接続
- 一度設定すれば複数パイプラインで再利用可能

---

# 接続の仕組み

```
CodePipeline
     ↓
CodeStar Connections
     ↓
GitHub（OAuth認証）
     ↓
リポジトリにアクセス
```

---

# 接続の作成手順

1. AWSコンソールで「CodePipeline」を開く
2. 左メニュー「設定」→「接続」
3. 「接続を作成」をクリック
4. プロバイダー「GitHub」を選択
5. 接続名を入力

---

# 接続名の例

```
github-connection
```

または

```
my-github-connection
```

わかりやすい名前をつけましょう

---

# GitHub認証フロー

1. 「GitHubに接続」をクリック
2. GitHubのログイン画面が表示
3. 「Authorize AWS Connector」をクリック
4. アクセス許可を確認して承認

---

# アクセス許可の内容

- リポジトリの読み取り
- Webhookの作成
- コミットステータスの更新

**書き込み権限は含まれません**（安全）

---

# 接続ステータス

| ステータス | 意味 |
|-----------|------|
| 保留中 | 認証待ち |
| 利用可能 | 接続完了 |
| エラー | 認証失敗 |

「利用可能」になれば成功です

---

# 接続の確認方法

1. 「設定」→「接続」を開く
2. 作成した接続を選択
3. ステータスが「利用可能」か確認
4. 接続ARNをメモしておく

---

# よくあるエラー

**「認証に失敗しました」**
→ GitHubに再ログインして再試行

**「権限がありません」**
→ リポジトリのオーナーか確認

**「接続がタイムアウト」**
→ ブラウザのポップアップを許可

---

# セキュリティのポイント

- 接続は**最小権限**で作成される
- 不要になったら接続を**削除**する
- 定期的に接続を**確認**する

---
