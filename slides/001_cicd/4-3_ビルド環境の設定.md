# ビルド環境とは

**ビルドが実行されるDockerコンテナ**

- AWSが提供するマネージドイメージ
- または独自のDockerイメージ
- ビルドごとにクリーンな環境

---

# コンピューティングタイプ

| タイプ | vCPU | メモリ | 用途 |
|--------|------|--------|------|
| small | 3 | 3GB | 軽量ビルド |
| medium | 7 | 15GB | 一般的なビルド |
| large | 15 | 72GB | 大規模ビルド |

---

# どれを選ぶべきか

**small（推奨）**
- 静的サイト、Node.js、Python
- 学習用途

**medium**
- Java、大きなnpm依存

**large**
- Dockerビルド、並列テスト

---

# マネージドイメージ

AWSが提供する公式イメージ

- **Amazon Linux 2**: 汎用
- **Ubuntu**: 幅広い言語サポート
- **Windows Server**: .NET開発

---

# イメージの選択

```
aws/codebuild/amazonlinux2-x86_64-standard:5.0
```

- Node.js, Python, Java, Go などプリインストール
- 最新バージョンを使用推奨

---

# ランタイムバージョン

```yaml
phases:
  install:
    runtime-versions:
      nodejs: 18
      python: 3.11
```

複数言語を同時に指定可能

---

# 利用可能なランタイム

| 言語 | バージョン例 |
|------|-------------|
| Node.js | 16, 18, 20 |
| Python | 3.9, 3.10, 3.11 |
| Java | corretto11, corretto17 |
| Go | 1.20, 1.21 |

---

# 環境変数の設定方法

**1. CodeBuildコンソールで設定**
- プロジェクト作成時に指定

**2. buildspec.ymlで設定**
- env セクションに記述

**3. Parameter Store連携**
- 機密情報はこちらを推奨

---

# コンソールでの設定

1. CodeBuildプロジェクトを開く
2. 「編集」→「環境」
3. 「追加設定」を展開
4. 「環境変数」に追加

---

# 機密情報の扱い

```yaml
env:
  parameter-store:
    API_KEY: /myapp/api-key
  secrets-manager:
    DB_PASS: arn:aws:secretsmanager:...
```

**平文で書かない！**

---

# 実践：ビルドプロジェクト作成

1. CodeBuildコンソールを開く
2. 「ビルドプロジェクトを作成」
3. 環境設定を選択
4. buildspec.ymlを指定

---

# まとめ

- ビルド環境はDockerコンテナで実行

- マネージドイメージが便利
