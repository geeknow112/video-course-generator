# CodePipelineとは

**継続的デリバリーサービス**

- コードの変更を自動的に検知
- ビルド・テスト・デプロイを自動実行
- 複数のAWSサービスと連携

---

# パイプラインの流れ

```
ソース → ビルド → テスト → デプロイ
  ↓        ↓        ↓        ↓
GitHub   CodeBuild  CodeBuild  S3/Lambda
```

各ステージを自由に組み合わせ可能

---

# 構成要素①：パイプライン

- 全体の流れを定義
- 複数のステージで構成
- 1つのパイプライン = 1つのワークフロー

---

# 構成要素②：ステージ

- パイプライン内の処理単位
- 例：Source、Build、Deploy
- ステージ間は順番に実行

---

# 構成要素③：アクション

- ステージ内の具体的な処理
- 例：GitHubからコード取得、S3にデプロイ
- 並列実行も可能

---

# 構成要素④：アーティファクト

- ステージ間で受け渡すファイル
- S3に自動保存される
- ビルド成果物、設定ファイルなど

---

| 項目 | 料金 |
|------|------|
| アクティブなパイプライン | $1/月 |
| 最初の1つ | **無料** |

※30日間使用しないパイプラインは非アクティブ扱い

---

# 他ツールとの比較

| ツール | 特徴 |
|--------|------|
| CodePipeline | AWS統合、シンプル |
| Jenkins | 高機能、自己管理必要 |
| GitHub Actions | GitHub統合、YAML定義 |
| CircleCI | クラウド型、高速 |

---

# CodePipelineを選ぶ理由

- AWSサービスとの**ネイティブ連携**
- IAMによる**細かい権限管理**
- **サーバーレス**で運用負荷ゼロ
- 最初の1パイプラインは**無料**

---

# まとめ

- CodePipelineは継続的デリバリーサービス
- パイプライン → ステージ → アクションの階層構造
- 最初の1つは無料で使える
- AWSサービスとの連携が強み
