---

# なぜ環境変数が必要か

**コードに直接書いてはいけないもの**

- APIキー
- データベース接続情報
- 外部サービスの認証情報
- 環境ごとの設定値

---

# 環境変数のメリット

- **セキュリティ**: 機密情報をコードから分離
- **柔軟性**: 環境ごとに値を変更可能
- **保守性**: 設定変更にデプロイ不要

---

# Lambdaでの環境変数設定

**コンソールから設定**

1. Lambda関数を開く
2. 「設定」タブ → 「環境変数」
3. 「編集」をクリック
4. キーと値を入力
5. 「保存」

---

# コードからの参照

```javascript
exports.handler = async (event) => {
  // 環境変数を取得
  const apiKey = process.env.API_KEY;
  const dbHost = process.env.DB_HOST;
  
  // 使用
  console.log(`Connecting to ${dbHost}`);
  
  return { statusCode: 200 };
};
```

---

# 環境変数の暗号化

**KMSで暗号化可能**

- 保存時に自動暗号化
- 実行時に自動復号
- カスタムKMSキーも使用可能

機密情報はSecrets Managerを推奨

---

# Secrets Managerとは

**機密情報の安全な管理サービス**

- 自動ローテーション
- アクセス制御
- 監査ログ
- バージョン管理

---

# Secrets Managerの使い方

```javascript
const { SecretsManager } = require('@aws-sdk/client-secrets-manager');

const client = new SecretsManager();

const response = await client.getSecretValue({
  SecretId: 'my-api-key'
});

const secret = JSON.parse(response.SecretString);
```

---

# 環境変数 vs Secrets Manager

| 項目 | 環境変数 | Secrets Manager |
|------|----------|-----------------|
| 設定 | 簡単 | やや複雑 |
| ローテーション | 手動 | 自動可能 |
| 料金 | 無料 | 有料 |
| 用途 | 一般設定 | 機密情報 |

---

# CI/CDでの環境変数管理

**buildspec.ymlでの設定**

```yaml
env:
  variables:
    NODE_ENV: production
  parameter-store:
    DB_PASSWORD: /myapp/db/password
  secrets-manager:
    API_KEY: my-api-secret:api_key
```

---

# Parameter Storeの活用

**Systems Manager Parameter Store**

- 無料で使える
- 階層構造で管理
- 暗号化オプションあり

```
/myapp/dev/db_host
/myapp/prod/db_host
```

---

# ベストプラクティス

- 機密情報はコードにハードコードしない
- 環境ごとに異なる値を使用
- 本番の機密情報はSecrets Manager
- 一般設定はParameter Store
