# トラブルシューティング

## よくある問題と解決方法

---

# 問題1: テストが開始されない

**原因と対策:**

| 原因 | 確認・対策 |
|------|------------|
| サブネット設定 | インターネットアクセス可能か |
| NAT Gateway | プライベートサブネットの場合必須 |
| セキュリティグループ | アウトバウンド443許可 |

**確認方法:**
CloudWatch Logs → `/ecs/distributed-load-testing`

---

# 問題2: 負荷が上がらない

**原因と対策:**

| 原因 | 確認・対策 |
|------|------------|
| タスクリソース不足 | CFnパラメータでサイズ変更 |
| ターゲット側の制限 | WAF/API Gatewayスロットリング確認 |
| ネットワーク帯域 | 大量データ転送時の制約 |

---

# 問題3: エラー率が高い

**原因と対策:**

| 原因 | 対策 |
|------|------|
| システムの限界 | 正常 → 改善ポイント特定 |
| タイムアウト設定 | JMeterでタイムアウト値調整 |
| スクリプトのバグ | URL/ヘッダー設定を確認 |

---

# ログ確認方法

**CloudWatch Logs:**
```
ロググループ: /ecs/distributed-load-testing

確認内容:
- エラーメッセージ
- 接続失敗の原因
- タイムアウト発生
```

---

# トラブルシューティングのコツ

1. **まずログを見る**
   → エラーメッセージに原因あり

2. **小さく始める**
   → 1タスク/少ユーザーで動作確認

3. **設定を見直す**
   → VPC/サブネット/SG
