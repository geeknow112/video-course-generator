# DLTとは

## AWS Distributed Load Testing

AWSが提供する負荷テストソリューション

---

# DLTの特徴

**🚀 サーバーレス**
ECS Fargateで負荷生成、サーバー管理不要

**📈 スケーラブル**
タスク数を増やすだけで数千〜数万同時接続

**🔧 JMeter対応**
既存スクリプトをそのまま使用可能

**🖥️ Webコンソール付き**
テスト作成・実行・結果確認がGUIで完結

---

# DLTアーキテクチャ

```
┌─────────────────────────────────────────────────────┐
│                  ユーザー                            │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│  CloudFront → S3 (Webコンソール)                     │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│  API Gateway → Lambda (テスト管理)                   │
│                  ↓                                   │
│              DynamoDB (設定・結果保存)               │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│  Step Functions → ECS Fargate (負荷生成)            │
│                   (Taurus/JMeter)                    │
└─────────────────────────────────────────────────────┘
```

---

# 使用するAWSサービス

| サービス | 役割 |
|----------|------|
| CloudFront + S3 | Webコンソールホスティング |
| API Gateway + Lambda | APIバックエンド |
| DynamoDB | テスト設定・結果の保存 |
| Step Functions | テスト実行オーケストレーション |
| ECS Fargate | 負荷生成タスク実行 |
| CloudWatch | ログ・メトリクス収集 |

---

# 従来の負荷テスト vs DLT

| 項目 | 従来 | DLT |
|------|------|-----|
| 環境構築 | 数日〜数週間 | **15分** |
| サーバー管理 | 必要 | **不要** |
| スケール | 手動で増設 | **タスク数変更のみ** |
| コスト | 常時稼働 | **使った分だけ** |

---

# まとめ

- DLTはAWS公式の負荷テストソリューション
- サーバーレスでスケーラブル
- JMeterスクリプトがそのまま使える
- Webコンソールで簡単操作
