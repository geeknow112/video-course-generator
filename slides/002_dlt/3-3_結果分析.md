# テスト結果の分析

## 重要なメトリクスの読み方

---

# 主要メトリクス

| メトリクス | 説明 |
|------------|------|
| Average Response Time | 平均レスポンスタイム |
| p50 (Median) | 50%のリクエストがこの時間内 |
| p90 | 90%のリクエストがこの時間内 |
| p99 | 99%のリクエストがこの時間内 |

---

# なぜパーセンタイルが重要？

```
平均: 100ms
でも...

99%のリクエスト: 50ms
1%のリクエスト: 5000ms (5秒!)

→ 100人に1人が5秒待たされる
```

**p99を監視することが重要！**

---

# その他のメトリクス

| メトリクス | 説明 |
|------------|------|
| Requests per Second | スループット（処理能力） |
| Success Count | 成功リクエスト数 |
| Error Count | エラーリクエスト数 |
| Bandwidth | データ転送量 |

---

# グラフの見方

**レスポンスタイム推移**
- 負荷増加に伴う変化を確認
- 急上昇ポイント = システムの限界

**エラー率推移**
- エラー発生タイミングを特定
- 限界を超えた負荷を把握
