## 台本

### なぜJMeterを使うのか（0:00-2:00）

これまでは、Webコンソールから簡単なHTTPテストを作成しました。
しかし、実際のシステムでは、より複雑なシナリオが必要になることがあります。

例えば：
- ログインしてから操作を行うシナリオ
- 複数のAPIを順番に呼び出すシナリオ
- 特定のパターンでリクエストを変化させるシナリオ

このような複雑なテストを行うために、
JMeterスクリプトを使用します。

---

### JMeterとは（2:00-4:00）

JMeterは、Apache Software Foundationが提供する
オープンソースの負荷テストツールです。

1998年からの長い歴史があり、
世界中で広く使われています。

主な特徴：
- GUIでテストを設計可能
- HTTPだけでなく、データベースやFTPなども対応
- プラグインで機能拡張可能
- XMLベースのテストスクリプト

DLTでは、JMeterで作成したスクリプトを
そのままアップロードして実行できます。

---

### JMeterの基本構成（4:00-7:00）

JMeterスクリプトの基本的な構成要素を説明します。

**Test Plan（テストプラン）**
テスト全体の設定を定義する最上位の要素です。

**Thread Group（スレッドグループ）**
仮想ユーザーの設定です。
ユーザー数、ランプアップ時間、繰り返し回数などを設定します。

**Sampler（サンプラー）**
実際のリクエストを定義します。
HTTP Request Samplerが最もよく使われます。

**Listener（リスナー）**
テスト結果を収集・表示します。
ただし、DLTでは結果はWebコンソールで確認するため、
リスナーは不要です。

**Config Element（設定要素）**
共通の設定を定義します。
HTTP Header Managerで共通ヘッダーを設定したり、
CSV Data Set Configでテストデータを読み込んだりします。

---

### DLTでのJMeter使用（7:00-10:00）

DLTでJMeterスクリプトを使う場合の流れは以下の通りです。

1. JMeter GUIでスクリプトを作成（.jmxファイル）
2. DLTのWebコンソールでスクリプトをアップロード
3. Task CountとConcurrencyを設定
4. テスト実行

注意点として、
JMeterスクリプト内のThread Group設定と、
DLTの設定が組み合わされます。

DLTでは、各Fargateタスクが1つのJMeterインスタンスを実行し、
Task Countでインスタンス数を制御します。

次のレクチャーでは、
実際にJMeterスクリプトを作成していきます。
